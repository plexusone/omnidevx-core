## [v0.2.0] - 2026-07-19

### Highlights

- Period aggregation: `DeveloperPeriodReport` (omnidevx.developer-period/v1) built from stored events via a daily-summary-then-rollup pipeline
- Identity resolution: canonical personId from GitHub usernames, hashed git emails, and device-scoped local accounts

### Added

- `report` package: `BuildDaily`/`Rollup`/`Build` reduce canonical events into a `DeveloperPeriodReport` with combined + bySource metrics, session dedup across day boundaries, coverage scoring, and a warning for period-total snapshot events not yet merged into daily buckets ([`fb6b59a`](https://github.com/plexusone/omnidevx-core/commit/fb6b59a))
- `identity` package: `Person`/`Identity`/`Map` resolving GitHub usernames, hashed git emails, and device-scoped local accounts to a canonical personId; `NewMap` rejects an identity claimed by two different people rather than silently picking one ([`91188e5`](https://github.com/plexusone/omnidevx-core/commit/91188e5))

### Changed

- `report`: use `slices.Sort` instead of `sort.Slice` for collection-mode ordering ([`5e4d54a`](https://github.com/plexusone/omnidevx-core/commit/5e4d54a))

### Documentation

- Concept pages for period reports and identity resolution, added to the MkDocs nav ([`1d50b43`](https://github.com/plexusone/omnidevx-core/commit/1d50b43))
- README: document `store`, `providers/git`, `providers/genericotel`, `identity`, and `report` packages (the Layout section had predated all but the original two); add a period-reports usage example ([`ff78f69`](https://github.com/plexusone/omnidevx-core/commit/ff78f69))

## [v0.1.0] - 2026-07-18

### Highlights

- Canonical event model and Collector contract for developer-experience telemetry (OmniDevX domain)
- Local JSONL event store: idempotent, inspectable, reprocessable
- Three collectors: Claude Code session history, git commits with AI co-author attribution, and a live OTLP/JSON metrics receiver
- Metadata-only privacy model enforced structurally and by tests: no prompt text, responses, or file contents are ever captured

### Added

- Canonical `Event` IR (omnidevx.event/v1): deterministic IDs, ai./devx. event-type namespaces, shared attribute keys, provenance with collection mode and confidence ([`d3c192d`](https://github.com/plexusone/omnidevx-core/commit/d3c192d))
- `Collector` interface with `CollectRequest`/`CollectionResult`, half-open `Period` semantics, and diagnostics instead of silent drops ([`d3c192d`](https://github.com/plexusone/omnidevx-core/commit/d3c192d))
- `store`: daily JSONL files per source under ~/.plexusone/omnidevx/data with owner-only permissions, ID-deduplicated idempotent writes, period/product-scoped reads, and damaged-line recovery ([`3639247`](https://github.com/plexusone/omnidevx-core/commit/3639247))
- `providers/claudecode`: stdlib-only importer for ~/.claude/projects session JSONL — prompts (sidechains excluded), assistant messages with model and token usage, tool completions via tool_use/tool_result correlation, session spans ([`aebb0f3`](https://github.com/plexusone/omnidevx-core/commit/aebb0f3))
- `providers/git`: devx.change.committed events built on github.com/grokify/gogit, with change stats, author identity, commit-hash IDs that deduplicate clones, and AI attribution from Co-authored-by trailers via the canonical KnownAITools registry ([`89f6ec0`](https://github.com/plexusone/omnidevx-core/commit/89f6ec0))
- `providers/genericotel`: OTLP/HTTP (JSON) receiver converting Claude Code and Codex CLI token/cost metrics to ai.usage.recorded events with otel-mode provenance; log payloads acknowledged and discarded ([`ad56256`](https://github.com/plexusone/omnidevx-core/commit/ad56256))

### Documentation

- README with OmniDXI disambiguation and usage ([`250bb0d`](https://github.com/plexusone/omnidevx-core/commit/250bb0d))
- MkDocs site: canonical events, event store, privacy model, and per-provider guides including Claude Code OTel setup ([`453f76a`](https://github.com/plexusone/omnidevx-core/commit/453f76a))
- Status badges ([`a5b30da`](https://github.com/plexusone/omnidevx-core/commit/a5b30da))

### Tests

- Contract, store, and provider tests on fabricated fixtures only: period boundaries, JSON round-trips, store idempotency and damaged-line recovery, per-provider parsing, OTLP ingestion idempotency, and privacy assertions that content-like attribute keys fail the build ([`02227d1`](https://github.com/plexusone/omnidevx-core/commit/02227d1))

### Infrastructure

- CI workflows (go-ci, go-lint, go-sast-codeql), golangci-lint config, dependabot, MIT license, gitignore ([`67f5fce`](https://github.com/plexusone/omnidevx-core/commit/67f5fce))
